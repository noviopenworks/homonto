# Onto Init Implementation Plan

> Implement task-by-task with TDD.

## Global Constraints (Design Doc + delta spec)

- `onto init` scaffolds `docs/{changes,specs,adr,guides}` idempotently (never
  overwrite), reports created vs skipped, `--dir` flag default ".".
- Framework-install GATE runs first; on ANY failure: print specific guidance,
  exit non-zero, and write NOTHING under `docs/`. Gate order: (1) no homonto.toml
  → "run homonto init"; (2) no `[frameworks.onto]` → "declare [frameworks.onto]
  and run homonto apply"; (3) declared but no `.homonto/catalog/skills/onto/` →
  "run homonto apply".
- Gate reads homonto.toml with `go-toml/v2` (minimal `struct{ Frameworks
  map[string]any `toml:"frameworks"` }`) — do NOT use `internal/config.Load`, do
  NOT construct engine/adapters. `onto` stays isolated from homonto's
  projection pipeline (imports none of internal/{cli,engine,config,adapter,catalog}).
- Additive only; no change to onto status / ontostate / homonto.
- Gates: `go build ./...`, `go test [-race] ./...`, `go vet`, `gofmt -l .`, `go
  mod tidy` clean; both binaries build.
